What this integration does
  1. Mic gate. SAA's prediction.cls === 2 drives conversation.setMicMuted(false/true) so the agent only hears you when SAA believes you're talking to the device.
  2. Self-trigger suppression. When EL's onModeChange fires speaking, SAA enters markResponding(true) so it doesn't classify the agent's own TTS as a new user turn.
  3. Contextual updates. SAA speechReady, face-count changes, and sustained human-directed speech each fire conversation.sendContextualUpdate(...) so the agent's prompt is enriched with attentional signal in real time.
  4. Client tools. The agent can call get_user_attention, get_face_count, and get_last_directed_utterance to introspect the room without spending audio tokens.
